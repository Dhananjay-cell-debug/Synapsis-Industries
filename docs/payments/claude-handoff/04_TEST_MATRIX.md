# Test Matrix

## Local Automated Tests

Minimum checks before deployment:

- `npx tsc --noEmit --pretty false`
- Unit tests or route-level tests for:
  - INR Razorpay order creation
  - USD Razorpay order creation
  - existing pending row reuse with same currency
  - pending row currency mismatch
  - verify route amount mismatch rejection
  - webhook idempotency

## Manual Browser Tests

### Indian Domestic Deal

Setup:

- `clientCountry = "IN"`
- `currency = "INR"`
- `paymentProvider = "razorpay"`
- `acceptInternationalCards = false`

Expected:

- Client UI shows INR.
- Razorpay order response returns `currency: "INR"`.
- Domestic NEFT option remains available where intended.
- Payment row stores `currency = INR`, `provider = razorpay`.

### International USD Deal

Setup:

- `clientCountry = "US"` or another non-India country.
- `currency = "USD"`
- `paymentProvider = "razorpay"`
- `acceptInternationalCards = true`

Expected:

- Client UI shows USD.
- Razorpay order response returns `currency: "USD"`.
- Domestic NEFT option is hidden.
- Payment row stores `currency = USD`, `provider = razorpay`.
- Invoice and email show USD.

### Missing Country Conservative Path

Setup:

- `currency = "INR"`
- `acceptInternationalCards = true`
- `clientCountry` empty

Expected:

- Current behavior defaults to INR.
- Admin should be asked to set country before using international path.

## Real-Money International Test Guidance

Use a tiny live test only after the Phase 2 and Phase 4 gaps are fixed and deployed.

What will prove international payments:

- A low-value live transaction from a trusted foreign-issued card or actual overseas client.
- Razorpay Dashboard transaction shows international card/payment method and the charge currency.
- Synapsis payment row stores the same charge currency and amount.
- Invoice/email show the same charge currency and amount.
- Settlement report later confirms settlement currency, fees, taxes, and final amount.

What will not prove international payments:

- Paying from an Indian card while standing in India.
- Paying INR through domestic checkout.
- Seeing the INR 50,000 Dashboard transaction limit.

Suggested low-risk real test:

1. Create a dedicated internal test deal, not a real client deal.
2. Set total price low enough that the phase payment is within the INR 50,000 equivalent transaction limit.
3. Configure it as USD international Razorpay.
4. Ask a trusted overseas cardholder to pay the smallest acceptable phase amount.
5. Compare:
   - Synapsis payment row
   - Razorpay payment detail
   - Razorpay order detail
   - invoice generated by Synapsis
   - email notification
   - settlement once available

Do not use a live payment until the phase-tab bypass is fixed.

## Dashboard Checks

For every live test, record:

- Razorpay order id
- Razorpay payment id
- Synapsis deal token
- Synapsis payment phase
- charge amount
- charge currency
- payment method/card country
- settlement amount
- settlement currency
- fees and GST/tax
- FIRS/FIRC/export documentation availability if applicable
